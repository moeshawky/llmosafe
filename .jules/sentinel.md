## 2024-03-24 - [CRITICAL] Fix C-ABI Out-of-bounds Memory Read in llmosafe_calculate_halo
**Vulnerability:** The FFI function `llmosafe_calculate_halo` used an unbounded C-string null-terminator scan (`CStr::from_ptr`) to convert raw pointers from C into Rust strings. Without a length bound, if the caller passed a non-null-terminated string, Rust would scan memory indefinitely looking for a null byte, leading to out-of-bounds reads or segmentation faults.
**Learning:** In C-ABI bindings, relying solely on null-terminated strings (`CStr`) can lead to severe memory safety issues when bridging with external C/Python environments where memory layout might differ or null terminators might be omitted (like dynamically allocated buffers or substrings).
**Prevention:** Always require explicit length bounds alongside raw pointers in FFI functions, and use `core::slice::from_raw_parts` to construct bounded slices. This ensures Rust only reads memory within the designated bounds, avoiding unconstrained memory scans.
