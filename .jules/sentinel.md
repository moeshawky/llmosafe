## 2026-04-12 - [CRITICAL] Out-of-bounds Read in C-ABI String Parsing
**Vulnerability:** The `llmosafe_calculate_halo` C-ABI function previously used `core::ffi::CStr::from_ptr(text_ptr)` to parse a string from C. This implicitly relies on the unbounded C-string null-terminator scan, allowing an out-of-bounds memory read (and possible DoS or remote memory exposure depending on use cases) if an unterminated string is passed.
**Learning:** Using `CStr::from_ptr` requires unconditional trust that the caller has null-terminated the provided string. In restricted, safety-critical environments where strings can be arbitrary binary buffers, this is inherently dangerous.
**Prevention:** Always prioritize explicit length bounds in C-ABI strings by accepting a `size_t text_len` parameter alongside the pointer. Use `core::slice::from_raw_parts(text_ptr as *const u8, text_len)` to parse memory securely instead of depending on unbounded null-terminator scans. Ensure `cbindgen.toml` is configured with `usize_is_size_t = true` so the headers use `size_t`.
